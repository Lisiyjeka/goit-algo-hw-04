## Аналіз продуктивності сортувань: вставками, злиттям та Timsort

На базі наданого виводу бенчмарків порівняно три алгоритми: сортування вставками, сортування злиттям та вбудований у Python `sorted`/`list.sort()` (Timsort). Оцінювалась медіана часу з 5 виконаннь на наборах даних: випадкові, відсортовані, зворотно відсортовані, майже відсортовані (≈5% перестановок), для розмірів до 100 000 елементів.

### Ключові висновки дослідження:

#### 1. **Абсолютний лідер: Timsort**
- **У 180-260 разів** швидший за merge sort на масивах до 50,000 елементів
- **У 11-14 разів** швидший за insertion sort на всіх розмірах
- Найкраще масштабування, особливо на відсортованих даних

#### 2. **Insertion sort: неочікувано конкурентноздатний**
- Конкурує з merge sort на масивах до 50,000 елементів
- Ефективний на майже відсортованих даних
- Практична цінність: як частина гібридних алгоритмів

#### 3. **Merge sort: урок про константні фактори**
- Теоретична O(n log n) не гарантує високу практичну продуктивність
- Великі накладні витрати на рекурсію та копіювання масивів
- На 100,000 елементах масштабується краще (12-43× vs Timsort)

#### 4. **Практичні рекомендації для розробників:**
 **Для продакшн-коду:** Завжди використовуйте вбудовані бібліотеки`sorted()` або `.sort()`
 **Для навчання:** Вивчайте алгоритми для розуміння принципів
 **Для спеціалізованих задач:** Розглядайте власні реалізації лише за наявності конкретних вимог

#### 5. **Головний інсайд:**
**Адаптивність перемагає теоретичну складність.** Timsort, що враховує структуру вхідних даних, є ідеальним прикладом того, як теоретичні алгоритми повинні адаптуватися для практичного застосування.


#### 6. **Поєднання вставок і злиття в Timsort забезпечує значну практичну перевагу**: від десятків до сотень разів швидше за merge sort і суттєво швидше за insertion sort.
